## 🚀 About the Project

**Media Remote Control** is a next-generation browser utility that bridges the gap between your desktop browser and your mobile device. Whether you are watching Netflix from your bed, listening to Spotify while cooking, or playing YouTube on a secondary monitor, this project lets you control playback, volume, and tabs directly from your smartphone without ever needing to touch your mouse or keyboard.

Just scan the QR code generated by the browser extension and your mobile device instantly transforms into a feature-rich remote control.

---

## 💡 Why this project?

Watching media on a laptop or a desktop connected to a TV/monitor is a common scenario. However, having to physically walk to the computer to pause a video, change the volume, or switch tabs is an unnecessary friction. 

Existing solutions either require installing heavy desktop applications, being on the exact same local network, or are limited to specific platforms (like Spotify Connect). **Media Remote Control** solves this by leveraging modern WebSockets, cloud architecture, and a lightweight browser extension to provide universal media control for *any* website, anywhere.

---

## ✨ Key Features

- **📱 Instant Pairing via QR Code**: No accounts, no logins. Just open the extension, scan the QR code with your phone's camera, and you are connected.
- **🎧 Universal Media Control**: Play, pause, mute, and adjust volume for any media playing in your browser.
- **🎨 Real-Time Metadata**: View the live title, artist, album, duration, and rich media artwork directly on your remote.
- **📑 Tab Management**: See all open tabs playing media. Seamlessly switch your remote control context between different tabs.
- **🔖 Remote Bookmarking**: Found something interesting while sitting on the couch? Bookmark the active tab on your host browser with a single tap.
- **⚡ Blazing Fast**: Built on Fastify WebSockets and Upstash Redis for sub-millisecond latency.

## 🏗 Architecture & Codebase Overview

The repository is organized into a highly modular, monorepo-style structure containing three main components:

### 1. `extension` (Host Client)
A modern browser extension built using [WXT (Web Extension Tools)](https://wxt.dev/).
- **Tech Stack**: React 19, Tailwind CSS v4, Framer Motion, TypeScript.
- **Responsibilities**: Injects content scripts into active pages to read media state (using the Media Session API) and sends commands to DOM audio/video elements. Generates the QR code and manages WebSocket connections to the server via a background service worker/offscreen document.

### 2. `client` (Remote Control)
A mobile-first web application that acts as the remote interface.
- **Tech Stack**: Next.js 16 (App Router), React 19, Tailwind CSS, Zustand (state management), Reconnecting WebSocket.
- **Responsibilities**: Provides a stunning, app-like UI. Connects to the server, authenticates via the scanned QR code session ID, and sends commands (Play, Pause, Volume, Bookmark) to the host.

### 3. `server` (Signaling & Relay)
A highly scalable WebSocket backend.
- **Tech Stack**: Node.js, Fastify, `@fastify/websocket`, TypeScript, Upstash Redis.
- **Responsibilities**: Validates sessions, manages WebSocket pools, routes messages instantly between the Remote (`client`) and the Host (`extension`), and handles presence (detecting when devices go offline).

## ⬇️ Downloads & Installation (Regular Users)

Ready to take control? Get started in two simple steps:

### Step 1: Install the Browser Extension
*The extension will be available on the Chrome Web Store and Firefox Add-ons soon. Keep an eye out for the official links!*

### Step 2: Open the Remote
1. Click the **Media Remote Control** icon in your browser toolbar.
2. A QR code will appear.
3. Open your smartphone's camera and scan the QR code.
4. Your phone will open the web remote. You are now paired and ready to go!

## 💻 Development Guide (Developers)

Want to run the project locally or contribute? Follow these steps.

### Prerequisites
- [Node.js](https://nodejs.org/en/) (v20+ recommended)
- [npm](https://www.npmjs.com/) or [pnpm](https://pnpm.io/)
- A Redis instance (Local or [Upstash](https://upstash.com/))

### Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone https://github.com/jadhavsharad/Media-Remote-Control-v2.git
   cd Media-Remote-Control-v2
   ```

2. **Server Setup:**
   ```bash
   cd server
   npm install
   ```
   - Create a `.env` file in the `server` directory and add your Redis credentials:
     ```env
     PORT=8080
     UPSTASH_REDIS_REST_URL=your_redis_url
     UPSTASH_REDIS_REST_TOKEN=your_redis_token
     ```
   - Start the server:
     ```bash
     npm run dev
     ```

3. **Client Setup:**
   ```bash
   cd ../client
   npm install
   ```
   - Create a `.env` file in the `client` directory:
     ```env
     NEXT_PUBLIC_WS_URL=ws://localhost:8080
     ```
   - Start the Next.js app:
     ```bash
     npm run dev
     ```

4. **Extension Setup:**
   ```bash
   cd ../extension
   npm install
   ```
   - Create a `.env` file in the `extension` directory:
     ```env
     WXT_SERVER_URL=ws://localhost:8080
     WXT_CLIENT_URL=http://localhost:3000
     ```
   - Start the extension development server:
     ```bash
     npm run dev
     ```
   - Load the generated `.output` extension into your browser with Developer Mode enabled.
